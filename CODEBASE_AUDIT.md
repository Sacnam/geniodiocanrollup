2.3 Frontend Issues
Auth Token Refresh Race Condition
In 
frontend/src/services/api.ts
:

Issue: When 401 occurs, 
refreshToken()
 is called. If multiple requests fail simultaneously (e.g., loading a dashboard with 5 widgets), 5 refresh requests will be sent.
Risk: The first refresh invalidates the token, causing the other 4 to fail permanently, logging the user out.
Fix: Implement a "refresh promise lock" (singleton pattern for the refresh request).
Hardcoded URL Fallback
In 
frontend/src/services/api.ts
:

typescript
const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://localhost:8000';
Issue: Fallback to localhost is risky for production builds if env var is missed.
Fix: Fail build if VITE_API_URL is missing in production mode.
